# 스킬 품질 검증 가이드

새 스킬을 작성하거나 기존 스킬을 수정한 후, 표준에 반영하기 전에 다음 검증을 수행합니다.

---

## 1. Baseline 테스트 (Red)
- 스킬 **없이** 동일한 작업을 요청
- AI가 자연스럽게 하는 행동을 기록 (어떤 단계를 빼먹는가?)
- 이것이 "실패 케이스" — 스킬이 교정해야 할 행동

## 2. 스킬 적용 테스트 (Green)
- 스킬을 **적용한 후** 동일한 작업을 요청
- AI가 스킬의 모든 단계를 따르는지 확인
- Baseline에서 발견된 문제가 교정되었는지 확인

## 3. 합리화 저항 테스트 (Refactor)
- "이 단계는 불필요하지 않아?" 등 우회를 유도하는 프롬프트 테스트
- AI가 절차를 유지하는지, 합리화하여 건너뛰는지 확인
- 건너뛰면 → Anti-Rationalization 섹션 보강

## 4. 배포 체크리스트
- [ ] frontmatter description이 스킬 선택에 충분한 정보를 제공하는가?
- [ ] 각 단계의 완료 조건이 명확하고 검증 가능한가?
- [ ] Anti-Rationalization 섹션이 최소 2개 패턴을 포함하는가?
- [ ] 토큰 크기가 3,000 이내인가? (초과 시 분리)
- [ ] 다른 스킬과의 파이프라인 연결이 명확한가?

## 5. 사후 모니터링
- 스킬 적용 후 첫 2주간 실사용 결과를 `.ai/DECISIONS.md`에 기록
- "스킬이 무시된 경우", "단계가 건너뛰어진 경우"를 수집하여 스킬 개선에 반영
