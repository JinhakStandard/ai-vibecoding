# CLAUDE.local.md 가이드 및 템플릿

이 문서는 `CLAUDE.local.md` 파일의 용도, 작성 규칙, 그리고 템플릿을 제공합니다.

---

## 1. CLAUDE.local.md란 무엇인가

`CLAUDE.local.md`는 **개인 개발자가 자신의 로컬 환경에 맞게 Claude Code의 동작을 조정**하기 위한 파일입니다.

- **git에 커밋하지 않음** (`.gitignore`에 포함)
- 프로젝트의 공통 규칙(`CLAUDE.md`)을 개인 환경에 맞게 보완
- 팀원 간 개발 환경 차이를 해소

---

## 2. 오버라이드 우선순위

Claude Code는 다음 순서로 설정을 적용합니다. 뒤에 오는 파일이 앞의 설정을 오버라이드합니다.

```
1. JINHAK 전사 표준 (CLAUDE.md - 표준 저장소)
   ↓ 상속
2. 프로젝트 CLAUDE.md (각 프로젝트 루트)
   ↓ 오버라이드
3. CLAUDE.local.md (개인 로컬, git 제외)
```

| 수준 | 파일 | 범위 | git 관리 |
|------|------|------|---------|
| 전사 표준 | `CLAUDE.md` (표준 저장소) | 모든 프로젝트 공통 | O |
| 프로젝트 | `CLAUDE.md` (프로젝트 루트) | 해당 프로젝트 전체 | O |
| 개인 | `CLAUDE.local.md` (프로젝트 루트) | 로컬 환경 전용 | X |

---

## 3. 사용 시나리오

### 3.1 프로젝트 예외 설정

프로젝트 공통 규칙에서 벗어나는 로컬 환경 설정이 필요한 경우:

```markdown
## 로컬 환경 설정
- 개발 서버 포트: 4000 (기본 3000과 충돌로 변경)
- 로컬 DB: SQLite 사용 (PostgreSQL 대신)
```

### 3.2 개인 선호 설정

개발자 개인의 작업 스타일에 맞는 설정:

```markdown
## 개인 선호
- 코드 설명 시 영어 주석도 병행해줘
- 테스트 코드 작성 시 Given-When-Then 패턴 사용
- 커밋 전에 항상 타입 체크를 먼저 실행해줘
```

### 3.3 실험적 규칙

아직 팀에 공유되지 않은 새로운 패턴을 시험하는 경우:

```markdown
## 실험적 규칙
- React Server Components 패턴으로 새 컴포넌트 작성 시도
- Zod 스키마 기반 폼 검증 패턴 적용
```

## 4. 작성 규칙 및 금지 사항

### 4.1 작성 규칙

- **보완 목적만**: 프로젝트 `CLAUDE.md`에 없는 로컬 환경 정보를 추가
- **간결하게 작성**: 필요한 내용만 최소한으로 기록
- **이유 명시**: 왜 이 오버라이드가 필요한지 간단히 설명

### 4.2 오버라이드 불가 항목 (금지)

다음 규칙은 `CLAUDE.local.md`에서 **절대 오버라이드할 수 없습니다**:

| 분류 | 금지 사항 | 이유 |
|------|----------|------|
| 보안 | `.env` 커밋 허용 | 전사 보안 정책 위반 |
| 보안 | `--no-verify` 허용 | Git hooks 무력화 |
| 보안 | `push --force` 허용 | 코드 이력 파괴 위험 |
| 보안 | 하드코딩된 비밀키 허용 | Vault 정책 위반 |
| 품질 | `any` 타입 사용 허용 | TypeScript 타입 안전성 훼손 |
| 품질 | `console.log` 프로덕션 허용 | 코드 품질 기준 위반 |
| API | PUT/PATCH/DELETE 사용 허용 | HTTP API 규칙 위반 |

> 보안 및 핵심 품질 규칙은 전사 표준이며 개인이 오버라이드할 수 없습니다.
