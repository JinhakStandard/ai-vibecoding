새로운 AI 개발 세션을 시작합니다. 이전 작업 내용을 파악하고 현재 진행 상태를 확인합니다.

> **Agent Memory**: Claude Code의 프로젝트 메모리가 활성화되어 있으면 이전 세션의 컨텍스트를 자동으로 회상합니다. `.ai/` 파일은 팀원 간 공유와 Git 추적이 필요한 핵심 사항을 보조합니다.

## 실행 절차

### 1단계: 이전 작업 확인
다음 파일들을 순서대로 읽습니다:

1. **`.ai/CURRENT_SPRINT.md`** - 현재 진행 중/대기 중인 작업 파악
2. **`.ai/SESSION_LOG.md`** - 최근 세션 작업 확인 (최근 3개 세션)
3. **`.ai/DECISIONS.md`** - 최근 기술 의사결정 확인

파일이 없으면 `.ai/` 폴더 구조 생성을 제안합니다.

### 2단계: JINHAK 표준 버전 체크
1. 프로젝트 `CLAUDE.md`에서 `jinhak_standard_version` 값 확인
2. 표준 저장소(`jinhak_standard_repo`)의 CHANGELOG.md에서 최신 버전 확인
3. 버전이 다르면 변경 내역을 요약하여 업데이트 여부를 사용자에게 확인

### 3단계: 프로젝트 상태 파악
1. `git status`로 미커밋 변경사항 확인
2. `git log --oneline -5`로 최근 커밋 확인
3. 진행 중인 작업이 있으면 이어서 작업할지 확인

### 4단계: 세션 브리핑
사용자에게 다음 내용을 요약하여 보고합니다:

```markdown
## 세션 브리핑

### 이전 작업 요약
- [최근 세션에서 수행한 작업 요약]

### 현재 진행 상태
- 진행 중: [작업 목록]
- 대기 중: [작업 목록]
- 완료: [최근 완료된 작업]

### 미커밋 변경사항
- [있으면 파일 목록, 없으면 "없음"]

### JINHAK 표준 버전
- 현재 적용: vX.X
- 최신 버전: vX.X
- 상태: [최신 / 업데이트 필요]

---
오늘 작업할 내용을 알려주세요.
```

## .ai/ 폴더가 없는 경우
다음과 같이 안내합니다:
```
프로젝트에 .ai/ 폴더가 없습니다.
JINHAK 표준에 따라 세션 관리 구조를 생성하시겠습니까?

생성되는 파일:
- .ai/SESSION_LOG.md (세션 작업 기록)
- .ai/CURRENT_SPRINT.md (현재 작업 현황)
- .ai/DECISIONS.md (기술 의사결정)
- .ai/ARCHITECTURE.md (시스템 아키텍처)
- .ai/CONVENTIONS.md (코딩 컨벤션)
```

## PR 기반 세션 재개
`--from-pr` 옵션으로 특정 PR의 컨텍스트에서 세션을 시작할 수 있습니다:
```bash
claude --from-pr 123
```
이 경우 해당 PR의 변경사항과 코멘트를 자동으로 컨텍스트에 로드합니다.
